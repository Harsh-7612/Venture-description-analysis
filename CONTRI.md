# Contributing to Venture Description Analyzer

We welcome contributions to improve the venture clustering pipeline! 

## Workflow

1. **Fork the Repository:** Create your own fork of the project.
2. **Create a Branch:** `git checkout -b feature/your-feature-name`
3. **Set Up the Environment:** Install dependencies via `pip install -r requirements.txt`.
4. **Implement Changes:** Ensure your code adheres to PEP 8 standards and includes type hints. If you modify the clustering parameters (e.g., `min_topic_size` or the embedding model), please document the impact on output granularity.
5. **Commit:** `git commit -m "Add descriptive message of your changes"`
6. **Push and PR:** Push to your fork and submit a Pull Request against the `main` branch.

## Issue Tracking
If you encounter bugs or have feature requests (e.g., integrating new LLM backends for label generation), please open an issue with a detailed description and context.
